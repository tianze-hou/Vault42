## Wiki 页面约定

### Frontmatter（YAML）

所有 Wiki 页面都应有以下 frontmatter：

```yaml
---
type: entity | concept | source | comparison
tags: [标签1, 标签2]
sources: [来源文件名或URL]
created: YYYY-MM-DD
updated: YYYY-MM-DD
---
```

### 页面类型说明

| 类型 | 目录 | 命名格式 | 说明 |
|------|------|----------|------|
| `entity` | `Wiki/实体/` | `实体名.md` | 具体的人、组织、产品 |
| `concept` | `Wiki/概念/` | 概念名.md | 原理、方法、术语 |
| `source` | `Wiki/来源/` | `YYYY-MM-DD 标题.md` | 原始资料的摘要 |
| `comparison` | `Wiki/对比/` | `对比主题.md` | 跨来源对比分析 |

### 交叉引用规则

- 页面间用 Wiki-link：`[[页面名]]`
- 每个提到的实体/概念都应链接到对应页面
- 如果对应页面不存在，先创建再引用
- 来源摘要页顶部必须链接到对应 Raw 文件：`[[Raw/文件名]]`

### 创建时机规则（自下而上原则）

**核心思想**：Wiki 里的概念必须从多篇来源中自然浮现，不是对单篇文章的过度抽象。

| 页面类型 | 创建条件 |
|----------|----------|
| 实体页 | 同一实体在 **≥2 篇不同来源** 中被明确提及 |
| 概念页 | 同一概念在 **≥2 篇不同来源** 中被讨论 |
| 对比页 | 对比主题已在 **≥2 篇来源** 中出现不同立场或数据 |
| 来源摘要页 | **始终创建**，每篇原始资料都对应一个 |

**具体流程**：

1. **第 1 篇来源**：在来源摘要页列出该概念（但**不创建**独立页）
2. **第 2 篇来源**：检查是否已有来源页提到过该概念 → **是** → 创建；**否** → 继续等待
3. **跨来源识别**：读新文章时，主动回溯已有来源摘要，发现重叠 → 触发建页

## 工作流

### 1. Ingest（摄取新资料）

当用户说"摄取"、"处理"、"存入"时：

1. **扫描** `Raw/` 根目录下所有 `.md` 文件，这些就是新文件
2. **阅读** 新文件内容（含图片）
3. **讨论** 与用户确认关键要点
4. **创建** `Wiki/来源/YYYY-MM-DD 标题.md` 摘要页
5. **更新** 相关的实体页（不存在则创建）
6. **更新** 相关的概念页（不存在则创建）
7. **更新** `Wiki/index.md`：添加新页面到对应表格
8. **追加** `Wiki/log.md`：格式 `## [YYYY-MM-DD] ingest | 标题`
9. **归档** 原始文件：使用 Obsidian CLI 移动到 `Raw/YYYY-MM/` 目录

```bash
obsidian move file="原始文件名.md" to="Raw/YYYY-MM/原始文件名.md"
```

> **Obsidian CLI 前提**：Obsidian CLI（`obsidian` 命令）需已安装并在 PATH 中。
> 安装参考：https://help.obsidian.md/cli
>
> 如果 Obsidian CLI 不可用，可以手动在 Obsidian 内移动文件，或由用户自行执行归档步骤。
>
> 一次摄取可能触及 10-15 个 Wiki 页面，这是正常的。
>
> **归档说明**：Obsidian 会自动更新正文中 `[[...]]` wikilink 的引用，但 frontmatter 的 `sources:` 字段是纯文本，不会自动更新——这是预期行为，不影响使用。

### 2. Query（查询）

当用户提问时：

1. **阅读** `Wiki/index.md` 找到相关页面
2. **深入** 阅读相关页面及其链接
3. **综合** 生成回答，标注来源页面
4. **归档**（可选）：如果回答有价值，写入 `Wiki/对比/` 作为新页面
5. **追加** `Wiki/log.md`：格式 `## [YYYY-MM-DD] query | 问题简述`

### 3. Lint（健康检查）

当用户说"检查 Wiki"、"lint"时：

检查以下问题并逐一修复：

- [ ] 有无**互相矛盾**的说法
- [ ] 有无**孤儿页面**（没有任何入链）
- [ ] 有无**提到但缺页**的实体/概念
- [ ] 有无**过时信息**（被新资料推翻但未更新）
- [ ] `index.md` 是否与实际文件同步
- [ ] 有无值得新建的汇总/对比页

追加日志：`## [YYYY-MM-DD] lint | 发现 N 个问题`

## Index 维护规则

`Wiki/index.md` 的每一行格式：

```markdown
| [[Wiki/来源/2026-04-21 标题]] | 一句话简介 | 2026-04-21 |
```

每次 ingest 或创建新页面后必须更新 index。

---

## Log 格式

```markdown
## [YYYY-MM-DD] 操作类型 | 标题

- 操作说明 bullet 1
- 操作说明 bullet 2
```

操作类型固定为：`init` / `ingest` / `query` / `lint` / `update`

## 图片处理

- 原始资料中的图片应下载到 `Raw/assets/` 目录
- Wiki 页面引用图片用：`![[assets/图片名.png]]`
- AI 读取 Markdown 时先处理文本，再单独读取图片获取视觉信息
